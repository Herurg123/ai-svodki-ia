# Карта внедрения редакционной спецификации

Версия: 2026-07-12

Этот файл не заменяет `editorial-policy.md`. Он фиксирует, какие части репозитория должны быть обновлены, чтобы утверждённые правила реально исполнялись кодом.

## 1. Конфигурация

### `automation/config/site.json`

Добавить или синхронизировать:

- ссылку Дзена;
- целевые количества сюжетов;
- часовой пояс;
- время публикации;
- путь к `stories.json`;
- параметры короткого выпуска.

### `automation/config/editorial.json`

Использовать как источник точных значений:

- фразы короткого выпуска;
- блока Дзена;
- сноски Meta;
- лимитов;
- терминологии;
- списков компаний;
- ограничений картинки.

## 2. Research-промпт

### `automation/prompts/research_candidates.md`

Добавить:

- точное окно timestamps;
- полный список мировых игроков;
- Китай и Азию;
- developer tools;
- российские организации;
- запрет добивать количество старыми материалами;
- `time_precision`;
- заметки о региональных пробелах;
- мягкие цели вместо искусственных квот.

## 3. Editorial-промпт

### `automation/prompts/daily_digest.md`

Добавить:

- 6–12 как целевой диапазон;
- короткий выпуск при 1–5 сюжетах;
- точную фразу короткого выпуска;
- цели 5–8 мировых и 3–5 российских;
- 2–3 абзаца на новость;
- 4–6 выводов;
- кликабельный блок Дзена;
- правило Meta;
- префикс `Обновление:`;
- терминологию;
- практические последствия;
- `editorial_notes`;
- `stories`;
- полный четырёхблочный `image_prompt`.

## 4. Генератор

### `automation/scripts/generate_digest_preview.py`

Добавить:

- чтение `editorial.json`;
- нормализацию короткого выпуска;
- нормализацию блока Дзена;
- безопасную обработку видимых текстовых узлов Meta;
- добавление одной сноски;
- проверку `Обновление:`;
- создание `stories.json`;
- создание `editorial_notes`;
- мягкие diversity overrides;
- точное временное окно;
- validation report для каждой нормы.

## 5. Архив

### `automation/scripts/bootstrap_archive.py`

Добавить чтение:

- `automation/content/*/stories.json`;
- старого RSS как fallback;
- timestamps;
- статусов `new` / `update`;
- организаций;
- тем;
- ключевых слов;
- источников.

### `automation/scripts/validate_archive.py`

Проверять:

- уникальные URL;
- timestamps;
- обязательные поля story;
- корректные статусы;
- отсутствие `Meta*` в служебных полях;
- корректность обновлений.

## 6. Сборка сайта

### `automation/scripts/build_site.py`

Проверить, что:

- H1 и обложка добавляются сборщиком;
- `article_html` не переписывается;
- блок Дзена остаётся кликабельным;
- сноска Meta сохраняется после блока Дзена;
- `stories.json` копируется в content, но не публикуется как видимый блок;
- `editorial_notes` не попадают в статью и RSS.

## 7. Валидация сайта

### `automation/scripts/validate_site.py`

Добавить проверки:

- короткая фраза является первым элементом при `short_digest=true`;
- короткая фраза отсутствует при обычном выпуске;
- 1–5 сюжетов допускаются;
- 0 сюжетов запрещены;
- 2–3 абзаца на сюжет;
- 4–6 пунктов «Что это значит»;
- точный блок Дзена;
- первое видимое `Meta*`;
- последующие `Meta`;
- ровно одна сноска Meta;
- отсутствие `Meta*` в URL и служебных полях;
- префикс `Обновление:`;
- терминология;
- валидный `stories.json`.

## 8. Тесты

Добавить fixtures:

1. Обычный выпуск с 8 сюжетами.
2. Короткий выпуск с 3 сюжетами.
3. Выпуск без сюжетов, ожидается error.
4. Выпуск с Meta.
5. Выпуск без Meta.
6. Выпуск с обновлением.
7. Выпуск с diversity override.
8. Источник только с датой без времени.
9. Выпуск без российских сюжетов.
10. Выпуск с неправильным блоком Дзена.

## 9. Порядок внедрения

1. Commit спецификации и `editorial.json`.
2. Обновление промптов.
3. Обновление генератора и валидаторов.
4. Бесплатные fixture-тесты.
5. Editorial-only тест на сохранённом research.
6. Полный research + editorial preview.
7. Прямая генерация картинки.
8. Полный artifact-only end-to-end.
9. Только после снятия заморозки: main, schedule, commit и FTP.

## 10. Запрет тихих изменений

Любое изменение содержательного редакционного правила требует:

- обновления `editorial-policy.md`;
- обновления `editorial.json`, если правило машиночитаемо;
- обновления теста;
- отдельного описания в commit message.
