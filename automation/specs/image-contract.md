# Контракт изображения ИИ-Сводки

## Назначение

Этот контракт отделяет бесплатную техническую проверку изображения от платной
генерации и визуальной редакционной оценки.

Офлайн-этап не вызывает OpenAI, не обращается к сети, не меняет `posts/` и не
публикует RSS. Он создаёт детерминированную fixture-обложку и проверяет весь
технический путь до HTML и RSS.

## Технический контракт

- целевая модель будущего платного этапа: `gpt-image-2`;
- artifact-файл: `cover.png`;
- publish filename берётся из нормализованного `digest.cover_filename`;
- формат: PNG;
- размер: ровно 1536×864;
- соотношение: ровно 16:9;
- quality будущего API-запроса: `high`;
- PNG должен иметь корректные chunks и CRC;
- разрешены RGB и RGBA с bit depth 8;
- interlace запрещён;
- текстовые PNG chunks `tEXt`, `zTXt`, `iTXt` запрещены;
- после `IEND` не должно быть лишних байтов;
- SHA-256, размер и метаданные фиксируются в `image-manifest.json`.

## Prompt

`image_prompt` проверяется на четыре блока в точном порядке:

1. `Изображение 16:9:`;
2. `Главные визуальные темы:`;
3. `Композиция:`;
4. `Стиль:`.

Prompt должен содержать точный нормализованный заголовок выпуска. Проверка
также закрепляет ограничения, уже применяемые валидатором digest artifact.
Дополнительные формулировки из канонической спецификации пока выводятся как
предупреждения, чтобы не менять редакционное правило молча.

## Честные границы офлайн-проверки

Fixture PNG не пытается имитировать результат генеративной модели и специально
не содержит заголовок. Поэтому отчёт обязан содержать:

```json
{
  "visual_semantics_validated": false,
  "rendered_title_validated": false
}
```

Читаемость точного заголовка, отсутствие лишнего текста, логотипов, лиц и
водяных знаков проверяются отдельно после реального Image API preview.

## Режимы

### `offline_fixture`

- `network_used=false`;
- `openai_used=false`;
- используется только для бесплатного CI и artifact-only end-to-end.

### `image_api_preview`

- `network_used=true`;
- `openai_used=true`;
- создаётся отдельным request после успешного текстового preview и явного
  согласования пользователя;
- результат остаётся только в artifact до отдельного разрешения production.
