# Редакционная спецификация проекта «ИИ-сводки»

Версия: 2026-07-12  
Статус: утверждённая каноническая спецификация  
Язык публикации: русский

Этот документ является источником истины для research-промпта, editorial-промпта, Python-валидаторов, сборщика сайта, RSS и архива. Содержательные требования нельзя удалять, сокращать или заменять без отдельного согласования.

## 1. Архитектура

Рабочая цепочка:

1. GitHub-архив и `stories.json` задают контекст антидублей.
2. Research-этап собирает и проверяет кандидатов.
3. Editorial-этап выбирает сюжеты и создаёт статью.
4. Код нормализует технические метаданные.
5. Image API получает полный `image_prompt`.
6. Сборщик создаёт HTML, общий индекс и RSS.
7. Валидаторы проверяют выпуск.
8. Только после успешной проверки допускаются commit и FTP-деплой.

Blogger, Google Sheets и ручное копирование не являются частью production-архитектуры.

## 2. Временное окно

Основное окно поиска:

`published_at` предыдущего выпуска → плановое `published_at` текущего выпуска.

Для ежедневного выпуска это обычно интервал от 07:00 предыдущего дня до 07:00 текущего дня по `Europe/Moscow`.

Если источник показывает точное время, кандидат хранит:

- `published_at`;
- `published_date`;
- `time_precision: "datetime"`.

Если источник показывает только дату:

- `published_at: null`;
- `published_date`;
- `time_precision: "date"`.

Материал с точностью только до даты допускается по календарной дате и получает внутреннюю заметку о неполной точности времени.

## 3. Антидубли и обновления

Проверять архив за 7–14 дней.

Не повторять тему без существенного нового факта.

Сравнивать:

- смысл события;
- компанию или игрока;
- дату события;
- источники;
- URL;
- заголовки;
- записи `stories.json`;
- текущий `archive/index.json`.

Если тема уже была, но появился важный новый факт:

- `archive_status` должен быть `update`;
- заголовок сюжета обязан начинаться с `Обновление:`;
- новый факт должен быть явно описан;
- основной URL должен вести на материал об обновлении.

Для новой новости префикс `Обновление:` запрещён.

## 4. Количество сюжетов

Общая цель выпуска:

- 6–12 сюжетов.

Мягкие цели по разделам:

- мировые новости: 5–8;
- российские новости: 3–5.

Если достойных мировых или российских сюжетов меньше, слабые или старые материалы не добавляются ради количества.

Если найдено 1–5 достойных сюжетов:

- выпуск публикуется;
- `status` остаётся `ok`;
- `short_digest` устанавливается в `true`;
- первый элемент `article_html` должен быть:

```html
<p>День на новости выдался слабым - поэтому коротко</p>
```

- в `editorial_notes` добавляется запись `low_news_volume`.

Если не найдено ни одного достаточно подтверждённого сюжета:

- `status: "error"`;
- публикация не создаётся.

Нельзя включать формулировки вида «по компании новостей нет».

## 5. Разнообразие

Мягкие цели:

- не более двух выбранных сюжетов от одного издателя;
- не более двух выбранных сюжетов по одной компании или организации.

Превышение разрешено, если:

- события независимы;
- каждое событие имеет высокую значимость;
- причина сохранена в `selection.diversity_overrides`.

Без объяснения превышение считается ошибкой валидации.

Не пересказывать рекламные и телеграмные формулировки как факт.

Если утверждение подтверждено только лендингом, пресс-релизом или соцсетями, называть его продуктовым анонсом или заявлением компании, а не независимой оценкой.

Отделять факты от слухов, заявлений сторон и прогнозов.

Не включать шум, пустые инфоповоды и материалы без существенного факта.

## 8. Структура `article_html`

`article_html` не содержит:

- `<h1>`;
- `<img>`;
- `<html>`;
- `<head>`;
- `<body>`;
- `<script>`;
- `<style>`;
- `<iframe>`;
- Markdown;
- служебный текст.

`<h1>` и обложку добавляет сборщик.

Обычный выпуск начинается со вступления из 2–4 предложений.

Короткий выпуск начинается с точной фразы:

```html
<p>День на новости выдался слабым - поэтому коротко</p>
```

После неё идёт обычное вступление.

Разделы:

```html
<h2>Мировые лидеры ИИ</h2>
<h2>Российские лидеры ИИ</h2>
<h2>Что это значит</h2>
<h2>Все ИИ-Сводки</h2>
```

Раздел с российскими новостями допускается не выводить, если достойных российских сюжетов нет.

## 9. Структура каждой новости

Каждый сюжет содержит:

```html
<h3>Заголовок новости</h3>
<p>Первый содержательный абзац.</p>
<p>Второй содержательный абзац.</p>
```

Допускается третий абзац.

Итого: 2–3 содержательных абзаца на сюжет.

Объяснять:

- что произошло;
- что подтверждено;
- почему это важно;
- ограничения и неопределённости;
- применимые последствия для рынка;
- продуктов;
- исследований;
- бизнеса;
- автоматизации;
- маркетинга;
- разработки;
- создателей ИИ-контента.

Не требуется искусственно затрагивать все направления в каждой новости.

Источники ставятся только HTML-ссылками:

```html
<a href="URL">Название источника</a>
```

Markdown-ссылки запрещены.

## 10. Раздел «Что это значит»

Обязательная структура:

```html
<h2>Что это значит</h2>
<ol>
  <li>...</li>
</ol>
```

Количество выводов: 4–6.

Выводы должны вытекать из выбранных сюжетов, а не быть общими лозунгами.

## 11. Блок Дзена

В конце статьи обязателен точный блок:

```html
<h2>Все ИИ-Сводки</h2>
<p><a href="https://dzen.ru/suite/7971db4c-2a4e-449f-b8bf-c3907486d6f1">Архив ИИ-Сводок</a></p>
```

Заголовок `<h2>Все ИИ-Сводки</h2>` не изменять.

Текст ссылки должен быть `Архив ИИ-Сводок`.

## 12. Маркировка Meta

Правило применяется только к видимому тексту `article_html`.

Если статья упоминает Meta:

1. первое видимое упоминание должно быть `Meta*`;
2. все последующие видимые упоминания должны быть `Meta`;
3. в `stories.json`, `sources.json`, `topics`, URL, атрибутах HTML и служебных полях писать `Meta` без звёздочки;
4. после блока Дзена добавить ровно одну сноску:

```html
<p><em>*Meta и ее сервисы - в России запрещены</em></p>
```

Если Meta в статье отсутствует, сноска запрещена.

Нельзя ставить звёздочку у каждого упоминания.

## 13. Терминология и стиль

Писать:

- `ИИ`, а не `AI`;
- `агент ИИ`, а не `AI agent`;
- `LLM` без перевода;
- названия компаний и продуктов не переводить.

`AI` допускается внутри официального названия компании или продукта, например `Mistral AI`.

Стиль:

- ясно;
- делово;
- живо;
- без воды;
- без чрезмерного хайпа.

## 14. Технические метаданные

Код, а не модель, задаёт:

- `date`;
- `slug`;
- `title`;
- `published_at`;
- `author`;
- `cover_filename`.

Источник значений:

- `publication_date`;
- `site.json`;
- `editorial.json`;
- шаблон имени файла;
- часовой пояс;
- час публикации.

## 15. `editorial_notes`

`meta.json` содержит:

```json
{
  "editorial_notes": []
}
```

Допустимые типы заметок:

- `source_gap`;
- `low_news_volume`;
- `time_precision`;
- `regional_gap`;
- `diversity_override`;
- другие технические редакционные ограничения.

Эти заметки:

- сохраняются в GitHub;
- видны в artifact и Job Summary;
- не публикуются автоматически в статье;
- не попадают в RSS.

## 16. `stories.json`

Видимый CSV не публикуется.

Для каждого выпуска сохраняется `stories.json`.

Минимальная карточка сюжета:

```json
{
  "candidate_id": "cand-001",
  "section": "world",
  "headline": "Заголовок новости",
  "organization": "Meta",
  "topic": "генерация изображений",
  "event_type": "отключение функции",
  "status": "new",
  "summary": "Краткое описание",
  "published_at": "2026-07-11T10:30:00Z",
  "published_date": "2026-07-11",
  "time_precision": "datetime",
  "sources": [],
  "keywords": [],
  "geography": "world",
  "category": "multimodal"
}
```

`stories.json` используется для:

- антидублей;
- определения обновлений;
- анализа тем;
- контроля компаний;
- анализа издателей;
- построения `archive/index.json`;
- будущей статистики.

## 17. Промпт изображения

`image_prompt` должен быть на русском языке и содержать четыре смысловых блока:

1. тип изображения и точный заголовок;
2. главные визуальные темы;
3. композиция;
4. стиль и запреты.

Точный заголовок:

`ИИ-Сводка на D месяца YYYY`.

Постоянные ограничения:

- формат 16:9;
- без логотипов компаний;
- без дополнительного текста, кроме точного заголовка;
- без водяных знаков;
- без мелких интерфейсных надписей;
- без пустой половины кадра;
- без стокового корпоративного клипарта;
- без узнаваемых лиц реальных людей;
- нейтральные силуэты без идентифицируемых черт допустимы.

Полный `image_prompt` передаётся в Image API напрямую, без художественного переписывания и без программного наложения заголовка как основного режима.

## 18. Ошибки

`status: "error"` используется, если:

- не найдено ни одного достойного сюжета;
- невозможно подтвердить ключевые факты;
- нарушена строгая JSON-схема;
- ссылки не проходят проверку;
- итоговый HTML не проходит обязательную валидацию;
- сборка или RSS невалидны;
- изменён рабочий `posts/` во время preview.

Малое количество новостей само по себе ошибкой не является.

## 19. Что намеренно не возвращается из старой задачи

Не используются:

- Blogger как production-архив;
- Google Sheets;
- ручной HTML code block;
- текстовая строка `СТАТУС` внутри статьи;
- видимый CSV;
- некликабельный URL Дзена;
- ручное копирование;
- отдельная задача «ИИ-сводка — Лаборатория промпта».

Эквиваленты:

- `СТАТУС` → `digest.status` и `run-info.json`;
- CSV → `stories.json`;
- Blogger → GitHub content и `archive/index.json`;
- HTML для копирования → автоматически собранная страница.
