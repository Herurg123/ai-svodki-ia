# Статус внедрения редакционной спецификации

Дата: 2026-07-12  
Ветка назначения: `automation-prep`

## Реализовано

- Каноническая политика вынесена в `automation/config/editorial.json`.
- Research-промпт использует точное временное окно и расширенный список направлений.
- Editorial-промпт содержит короткий выпуск, Meta, обновления, блок Дзена, 2–3 абзаца и 4–6 выводов.
- Технические метаданные нормализуются кодом.
- `article_html` нормализуется по политике до валидации.
- Создаются `stories.json` и внутренние `editorial_notes`.
- Архив версии 2 читает структурированные карточки и поддерживает legacy RSS.
- Сборщик и валидатор сайта проверяют редакционную политику.
- Добавлены offline fixture и unit-тесты.
- Добавлен бесплатный workflow `Editorial policy check`.
- Добавлен контрактный валидатор, который не позволяет молча удалить ключевые правила из конфигурации и промптов.

## Не запускается этим внедрением

- OpenAI Responses API.
- Web search.
- Image API.
- FTP.
- Commit в `main`.
- Изменение рабочего `posts/` и RSS.

## Следующие контрольные этапы

1. Бесплатные GitHub Actions после загрузки пакета.
2. Editorial-only preview на сохранённом research с новым `request_id`.
3. Полный research + editorial preview на новой дате.
4. Прямая генерация обложки в artifact.
5. Полный end-to-end preview без изменения production.
6. Снятие заморозки и отдельное решение о production-деплое.
