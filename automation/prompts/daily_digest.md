## 1. Источники данных

Используй только сведения и URL из CANDIDATES_CONTEXT. На этом этапе не ищи новые материалы и не добавляй факты по памяти.

Не выбирай кандидатов с recommendation exclude. Не выбирай дубли. Кандидат с archive_status update допустим только тогда, когда в статье ясно описано существенное новое развитие.

## 2. Отбор

Выбери от {{MINIMUM_SELECTED_STORIES}} до {{MAXIMUM_SELECTED_STORIES}} наиболее значимых сюжетов, если исследовательский пул это позволяет.

Правила:

- не более двух выбранных сюжетов от одного издателя;
- не более двух сюжетов, связанных с одной компанией или организацией;
- сохрани разнообразие тем: модели, агенты, безопасность, исследования, инфраструктура, регулирование, роботы, корпоративные внедрения;
- включи достойные российские сюжеты, если в пуле есть российские кандидаты с significance_score не ниже 3 и recommendation include или consider;
- не заполняй выпуск слабыми материалами ради количества;
- если подходящих сюжетов меньше минимального числа, верни status error и кратко объясни причину.

## 3. Структура статьи

article_html должен содержать полный готовый текст статьи в чистом HTML.

Разрешены только теги:

- p
- h2
- h3
- ul
- ol
- li
- strong
- em
- a
- blockquote

У тега a разрешён только атрибут href.

Не добавляй в article_html:

- h1;
- обложку;
- html, head или body;
- script, style или iframe;
- Markdown;
- служебные комментарии;
- упоминания GitHub, RSS, Blogger, Дзена, автоматизации или внутреннего архива;
- отдельный раздел со списком источников.

Рекомендуемая структура:

1. Два или три вводных абзаца с главными темами выпуска.
2. <h2>Мировые лидеры ИИ</h2>
3. Для каждого мирового сюжета отдельный <h3>.
4. <h2>Российские лидеры ИИ</h2>, если выбраны российские сюжеты.
5. Для каждого российского сюжета отдельный <h3>.
6. <h2>Что это значит</h2>
7. Нумерованный список из четырёх–шести выводов.

Для каждой новости объясни:

- что произошло;
- что подтверждено;
- почему это важно;
- какие ограничения или неопределённости существенны.

Каждая новость должна содержать хотя бы одну ссылку вида:

<a href="https://точный-url-из-кандидата">Название источника</a>

Не изменяй URL и не добавляй URL, которых нет в CANDIDATES_CONTEXT.

## 4. Стиль

Пиши на русском языке. Стиль аналитический, ясный и профессиональный. Без рекламного восторга, канцелярита, неподтверждённых оценок и повторов. Не представляй заявление компании как независимо доказанный факт. Не копируй длинные фрагменты источников.

## 5. Описание и темы

description:

- от 150 до 300 символов;
- без HTML и ссылок;
- отражает главные темы выпуска.

topics:

- от 5 до 20 коротких тем;
- без дублей;
- только по выбранным сюжетам.

sources:

- содержит каждый URL, который присутствует в article_html;
- не содержит URL, отсутствующих в article_html;
- использует точные title, publisher и url из исследовательского пула.

## 6. Промпт обложки

image_prompt должен быть полноценным самостоятельным техническим заданием для прямой генерации изображения.

Он обязан иметь четыре смысловых абзаца в таком порядке:

1. «Изображение 16:9: ...» — назначение обложки и точный единственный читаемый заголовок «ИИ-Сводка на D месяца YYYY».
2. «Главные визуальные темы: ...» — конкретные сцены из выбранных новостей.
3. «Композиция: ...» — расположение центрального фокуса, окружающих сцен и нижней зоны.
4. «Стиль: ...» — палитра, детализация и запреты.

Требования:

- современная футуристическая редакционная обложка ежедневной ИИ-сводки;
- крупный и единственный читаемый текст — точный заголовок выпуска;
- от шести до десяти конкретных визуальных мотивов, если выпуск содержит достаточно сюжетов;
- без логотипов компаний;
- без лиц людей;
- без дополнительного текста;
- без водяных знаков;
- без мелких интерфейсных надписей;
- без технического имени файла;
- без пустой половины кадра;
- без стокового корпоративного клипарта.

## 7. Результат

Верни только строгий JSON по переданной схеме. Никакого Markdown, пояснений до JSON или текста после JSON.
